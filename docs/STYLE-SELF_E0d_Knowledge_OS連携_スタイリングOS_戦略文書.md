## 1. ★ ★ ★ ★ オーナー verbatim 記録(★ 改変禁止・全文保全)

### 1.1 目的

```
AI にセンスを丸投げするのではなく、良い服・良いスタイリング・良いブランドルック・良いムードボード分析・
失敗例・人間のレビューを Knowledge OS に蓄積し、それを次回以降のコーデ生成に反映させる。
```

### 1.2 重要な考え方

```
- AI は毎回ゼロから考えない
- Knowledge OS から関連する判断ルールを取得してから提案する
- 良い出力 / 悪い出力の評価ログを保存する
- 自分が実際に使う段階になったら、質の良い画像・ルック・ブランド資料・本の知識・スタイリング分析を
  入れて、コーデや服の理解を深めさせる
- Knowledge OS を「★ ★ AI のセンスを育てる ★ ★ 判断ルール工場」として扱う
```

### 1.3 コーデ生成前に取得したい 9 情報

```
1. 関連するファッション判断ルール
2. 似た世界観の過去ムードボード分析
3. 良かったコーデ出力例
4. 悪かったコーデ出力例
5. なぜ浅い / なぜ良いかのレビュー
6. 素材・色・シルエット・比率・重心・ラインの判断ルール
7. ブランドルックや参照画像から抽出したルール
8. 本から得た服・デザイン・身体・印象に関する知識
9. 人間レビューから得た改善ルール
```

### 1.4 保存したい 12 データ

```
- ムードボード分析結果
- コーデ提案
- 着用イメージ
- ユーザー評価
- 人間レビュー
- 良かった点
- 悪かった点
- 既視感があった理由
- 世界観が弱かった理由
- 次回使うべき判断ルール
- NG ルール
- 改善後の模範出力
```

### 1.5 今後の 10 ステップ生成フロー

```
1. ムードボードを分析する
2. 世界観コアを抽出する
3. Knowledge OS から関連判断ルールを取得する
4. 画像要素を日常服に翻訳する
5. 体型プロフィールと統合する
6. コーデを生成する
7. 着用イメージを生成する
8. 自己批評する
9. ユーザー / 人間レビューを保存する
10. 次回生成に反映する
```

### 1.6 ★ ★ ★ 特に重要(段階的アプローチ)

```
今はまだ本格的に質の高い情報を大量には入れていない。
ただし、今後自分で使う段階になったら、質の高いブランドルック・良いスナップ・良いスタイリング画像・
ファッション本の知識・失敗例・人間レビューを入れて、AI の服理解を深めさせる。
そのため、今の段階では ★ Knowledge OS 連携の ★ ★ 土台を作る。
完璧なセンスを今すぐ AI に求めるのではなく、★ ★ ★ 使えば使うほど判断ルールが蓄積され、
出力品質が上がる構造にする。
```

### 1.7 ★ ★ ★ ★ ★ 結論(★ 戦略の北極星)

```
STYLE-SELF は、AI が適当におしゃれを提案するアプリではなく、
★ ★ ★ Knowledge OS に蓄積された判断ルールを使って、世界観・体型・ムードボード・商品を統合する
```
