## 1. ★ ★ ★ ★ オーナー verbatim 記録(★ 改変禁止・全文保全)

### 1.1 根本方針変更

```
今のムードボード機能は、ムードボードを作ると長いプロンプトが自動生成され、それをAIに投げて長文回答が
返る構造になっている。
しかしこれは本質ではない。
私が作りたいのは「プロンプト生成ツール」ではなく、ChatGPT / Claude / Gemini のように、ムードボード・
画像・会話履歴・ユーザーの好みを保持しながら、対話で世界観を服・商品・着用イメージに変換していく
ファッションAI である。
```

### 1.2 現状の問題 7 項目

```
1. ムードボードが "文脈" ではなく "長いプロンプト" になっている
2. ユーザーがAIと対話して改善する体験になっていない
3. チャット履歴が残らないため、前回の好み・修正・判断が蓄積されない
4. 出力が長文で、ユーザーが服を選ぶ体験ではなく、AIの説明文を読まされる体験になっている
5. 画像生成も「世界観ルック」と「実商品試着」が混ざっていて、購入判断に使いづらい
6. Knowledge OSや過去の判断ルールが生成前に十分使われていない
7. ユーザーが「ここが違う」「もっとこうしたい」と言っても、次に継続される構造が弱い
```

### 1.3 作り直したい構造

```
ムードボードはプロンプトではなく、AI が参照する context object として保存する。
ユーザーには長いプロンプトを見せず、チャット画面で自然に会話させる。
```

### 1.4 必要な UX(13 項目)

```
- ChatGPT のようなチャット画面を中心にする
- ムードボードをチャットに添付できる
- ムードボード分析結果は裏側に保存する
- チャット履歴を永続保存する
- ユーザーの「好き」「違う」「もっと寄せる」「日常化する」「このアイテムだけ変える」を履歴として保存する
- 次回生成時に過去の会話・評価・Knowledge OS を参照する
- 長いプロンプトをユーザーに見せない
- ユーザーには「ムードボードから読み取った方向性」「提案コーデ」「修正ボタン」「画像」 のみを見せる
```

### 1.5 新しい 10 ステップ生成フロー

```
1. ユーザーがムードボードを作る
2. moodboard_analysis を作成して DB に保存
3. moodboard_core を抽出して保存
4. translation_rules を生成して保存
5. Knowledge OS から関連する判断ルールを取得
6. チャット履歴・ユーザー好み・体型情報と統合
7. AI がコーデ案を出す
8. ユーザーが対話で修正する
9. 修正内容を memory / feedback / judgment rule として保存
10. 次回以降の提案に反映する
```

### 1.6 画面構造

```
- 左: チャット履歴一覧
- 中央: AI スタイリストとのチャット
- 右 or 下: ムードボード、コーデ案、着用イメージ、商品候補
- 各チャットスレッドには:
  moodboard_id / user_profile_id / worldview_id / generated_outfits / feedback を紐づける
```

### 1.7 ★ ★ ★ ★ ★ 最重要宣言

```
STYLE-SELF は「ムードボードから長文プロンプトを作るアプリ」ではない。
「ムードボードを文脈として持った、対話型 AI スタイリスト」にする。

まずは既存の長文プロンプト生成 UI を主役から外し、ChatGPT / Claude / Gemini 型のチャット体験を
主役にする。
```

### 1.8 さらに強く

```
今の一番の間違いは、ユーザーがプロンプトを扱う設計になっていること。

ユーザーがやりたいのは:
- このムードボードの感じで服を考えて
- ちょっと違う
- もっと日常的に
- 靴は変えたい
- 自分が着たらどうなる?
- 似た商品ある?

だから、画面の本体はムードボード作成画面じゃなくて、AI スタイリストとの会話画面であるべき。

ムードボードは「入力」ではあるけど、主役じゃない。
主役は会話で世界観を服にしていく流れ。
```

### 1.9 Chat Thread 構造 9 要素

```
Chat Thread
├─ ユーザーの会話履歴
├─ 添付ムードボード
├─ ムードボード分析
├─ 世界観コア
├─ 体型プロフィール
├─ Knowledge OS から取得した判断ルール
├─ 生成したコーデ案
├─ 生成した画像
├─ 商品候補
└─ ユーザーのフィードバック

これが残る。
だから次にユーザーが「前のやつよりもっと不穏にして」と言ったら、AI が分かる。

今みたいに履歴が残らないと、毎回リセットされる。
それだと ChatGPT にも Claude にも負ける。
```

### 1.10 出力 UI 改善

```
現在のように、長い分析文・11項目・品質評価をすべて一括表示するのはやめる。

ユーザーにはまず短く、判断しやすい形で見せる。
```

表示順 7 項目:

```
1. 今回の方向性 1〜2文
2. コーデ案の要約
3. 具体アイテム一覧
4. ムードボード由来の要素
5. 画像
6. 修正ボタン
7. 詳細分析は折りたたみ
```

詳細分析(折りたたみ)5 項目:

```
- 参考画像の反映
- 11項目
- 品質評価
- Knowledge OS 参照ルール
- 生成プロンプト
```

### 1.11 ★ ★ ★ ★ ★ ★ ★ 結論

```
これはプロンプト改善の話ではなく、プロダクト構造の問題。

長いプロンプトを作って AI に投げる方式ではなく、チャット履歴・ムードボード・Knowledge OS・
ユーザーフィードバックを永続的な文脈として持つ、対話型 AI スタイリストに作り直す。

プロンプト生成アプリではなく、チャット型 AI スタイリストに設計を戻す。
```

## 11. 「ChatGPT / Claude / Gemini に負ける」発言の重み

- これは比喩ではなく ★ 競合定義。STYLE-SELF の競合は「ファッションアプリ」ではなく ★ 汎用対話 AI
- 汎用 AI に対する STYLE-SELF の優位は「文脈の専門性」=ムードボード・世界観コア・Knowledge OS・体型・購入判断を **永続文脈** として持つこと
- 文脈を捨てるワンショット構造のままでは、汎用 AI の方が「会話できる分」優れて見える → ★ 構造で負ける
- よって Chat Thread の永続化は「あれば良い機能」ではなく ★ ★ 競争上の生命線
