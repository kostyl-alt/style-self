## 1. ★ ★ ★ ★ オーナー verbatim 記録(★ 改変禁止・全文保全)

### 1.1 現状の問題

```
現在の画像生成は、世界観プロンプトから架空の人物・架空の服・架空の背景を生成しているため、
雰囲気確認には使えるが、★ ★ 購入判断に使えるリアル試着にはなっていない。
ユーザーが欲しいのは「AI が作ったファッション画像」ではなく、
★ ★ ★ 「自分がその実商品を着た時の判断材料」。
```

### 1.2 方針: 画像生成を 2 種類に分ける

#### 1.2.1 世界観ルック生成

```
★ 1. 世界観ルック生成
   - 目的:ムードボードの世界観やコーデの方向性を視覚化する
   - 架空モデルでも可
   - 購入判断ではなく、★ 発想・世界観確認用
   ★ ★ = 現状の C-1〜C-2g(product-to-model)がこれに該当
```

#### 1.2.2 ★ ★ ★ 実商品試着プレビュー(★ ★ ★ ★ 本命)

```
★ 2. ★ ★ ★ 実商品試着プレビュー(★ ★ ★ ★ 本命)
   - 目的:★ ★ 購入判断を助ける
   - ★ ユーザー写真または近い体型モデル画像を使う
   - ★ ★ 実在する商品画像を使う
   - FASHN などの Virtual Try-On API には ★ ★ ★ prompt だけではなく
     ★ model_image + garment_image を渡す
   - ★ ★ ★ 架空コーデではなく、★ ★ 選択された商品を着せる
```

### 1.3 ★ ★ ★ ★ ★ 重要

```
リアル試着は「世界観から生成された架空画像」では成立しない。
★ ★ ★ ★ 必ず ★ 実商品画像と人物画像を入力にする。
```

### 1.4 新しい 10 ステップフロー

```
1. ムードボード分析
2. Knowledge OS から判断ルール取得
3. コーデ提案
4. ★ 実在の商品候補を提示
5. ★ ★ ユーザーが商品を選択
6. ★ ★ ★ ユーザー写真または近い体型モデルを選択
7. ★ ★ ★ ★ FASHN の try-on API で着用プレビュー生成
8. 正面/斜め/横/後ろの複数アングルを表示
9. 丈感・肩幅・身幅・パンツ幅・靴とのバランスの確認メモを表示
10. 気になる点を対話で修正し、★ 類似商品を再提案
```

### 1.5 UI 変更

```
- 「世界観ルックを見る」
- ★ ★ 「この商品を自分で試す」
- ★ ★ 「近い体型モデルで見る」
の 3 ボタンに分ける。
```

### 1.6 注意

```
- 生成画像は実際の商品・サイズ・色味と異なる場合があると表示
- サイズ推薦は FASHN ではなく、★ 採寸情報・商品サイズ表・将来的な 3DLOOK/3D ボディスキャン等で別途対応
- ★ ★ 今の FASHN 画像はリアル試着ではなく、世界観ルックとして扱う
```

### 1.7 結論

```
STYLE-SELF のリアル試着の本命は、
★ ★ ★ ★ ★ 「世界観 → 実商品 → 自分/近い体型に着せる」
「世界観 → 架空画像」は ★ 購入判断用ではなく、★ ★ 発想確認用に限定する。
```

## 3. ★ ★ オーナー画像 verify による発見(★ 真因の構造)

### 3.1 ★ verify ケース(C-2g 後)

★ オーナーが C-2g(`e2f3583`)実装後に着用イメージ生成 →

### 3.2 ★ ★ ★ 真因(★ 設計レベル)

| レイヤー | 問題 |
|---|---|
| ★ FASHN モデル選択 | product-to-model = ★ ★ 服画像から ★ 架空人物を生成(★ 入力に「自分」がいない) |

| prompt 改善限界 | ★ E-0b-rev の 6 重視ポイントで詰めても ★ ★ 架空生成の枠を超えない |

→ ★ ★ ★ ★ ★ **真因 = ★ ★ ★ ★ FASHN モデル選択(product-to-model vs tryon-max)+ 入力データ(架空 vs 実物)**

### 3.3 ★ ★ ★ ★ 根本解決

```
[★ Before(E-0b-rev / C-2g)]
prompt(世界観 + 体型 + コーデ) → product-to-model → ★ 架空人物 + 架空服

         ↓ ★ ★ ★ 根本転換

[★ ★ After(E-0b-rev2 / Sprint G)]

    + (optional) prompt(質感補強)
```
