## 3. ★ 論点 2: 評価ロジック設計(★ 10 軸 / 6 チェック / 6 再生成条件)

### 3.1 ★ 10 評価軸の運用(★ E-0c §7.1 詳細化)

#### 3.1.1 ★ ★ ★ 推奨: ハイブリッド(★ スコア + チェック)

```
各 10 軸を 0-10 スコア(0=完全不在 / 5=平均 / 10=圧巻)で評価
+ 「10 番(1 点の強い違和感)」を ★ 特別扱い:
  - 0-2: 必ず不合格(★ ★ E-0c §1.3 #2「核アイテム必須」直結)
  - 3-5: 警告(妥協ライン)
  - 6-10: 合格
```

#### 3.1.2 ★ 合格判定基準(★ ★ 案)

| 判定 | 条件 |
|---|---|
| ★ 完全合格 | 合計 ≥ 70/100 + 10 番 ≥ 6 |
| 妥協合格 | 合計 ≥ 55/100 + 10 番 ≥ 3(★ N=2 失敗時の最後の砦)|
| ★ 不合格 | 合計 < 55 or 10 番 ≤ 2 |

#### 3.1.3 ★ ★ Sonnet エディタへの prompt 設計案

```
あなたは服好き / インフルエンサー視点のスタイリング編集者 AI です。
以下のコーデ提案を 10 評価軸で 0-10 スコアリングし、6 チェックで合否判定してください。

[評価軸]
1. 新規性: Pinterest 上位 100 件にない組み合わせか?
2. 既視感の少なさ: #ootd 海に埋もれず ★ スクロール止まるか?
... (10 軸全展開)

[出力 JSON]
{
  "scores": { "novelty": N, "rarity": N, ... },
  "total": N,
  "checks": { "shallow": "ok"|"ng", ... },
  "regenerate_reasons": [<6 条件のうち該当>],
  "verdict": "pass" | "compromise" | "regenerate",
  "improvement": "<不合格時の具体改善方向>"
}
```

### 3.2 ★ 6 チェックの運用(★ E-0c §1.3 #4 verbatim)

#### 3.2.1 ★ チェック内容

```
1. 服好き / インフルエンサーにとって ★ 浅くないか
2. ★ Pinterest の劣化版になっていないか
3. ★ ムードボードの本質を日常服に翻訳できているか
4. ★ 記憶に残る要素があるか
5. ★ 写真にした時に投稿したくなるか
6. ★ 商品として探せる粒度か
```

#### 3.2.2 ★ ★ 判定ルール(★ 推奨)

| ng 数 | 判定 |
|---|---|
| 0 | ★ 合格(他軸 OK の前提) |
| 1 | ★ ★ 妥協(警告付き)|
| 2 以上 | ★ ★ ★ 不合格(★ 再生成)|

### 3.3 ★ 6 再生成条件の運用(★ E-0c §1.3 #5 verbatim)

#### 3.3.1 ★ 検出方式

| 条件 | 検出方法 |
|---|---|
| 1. 一般的な黒モードに丸まり | ★ Sonnet 評価で「黒モード度」を判定(★ 9/10 で警告)|
| 2. 核アイテムなし | ★ 軸 10(1 点の強い違和感)が 0-2 → 自動該当 |
| 3. 参考画像本質 3 枚以下 | ★ buildMoodboardPrompt 参考画像枚数 N vs 反映数(Sonnet 評価)|
| 4. 新規性低 | ★ 軸 1 が 0-3 → 該当 |
| 5. 既視感強 | ★ 軸 2 が 0-3 → 該当 |
| 6. 着用イメージ綺麗すぎ | ★ ★ Vision 評価(★ C-2c-3 で対応)|

★ ★ → ★ ★ ★ いずれか該当で ★ ★ 再生成必須(★ E-0c verbatim 通り)
