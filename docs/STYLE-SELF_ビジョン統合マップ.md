## 0. 一文コンセプト

> **世界観から、服・商品・人・カルチャーに出会えるアプリ。**

補助定義:
- これは診断アプリでも、コーデアプリでも、服のECでもない。
- 「その人がなりたい印象・生き方・価値観・世界観を、
  服や物や文化に変換するアプリ」。
- パーソナルスタイル SNS 兼 コマース。ただし全てが「世界観」で繋がる。

最終目標(オーナーの言葉):
> 日本を変える。日本をファッションの街にする。
> 日本人が全員おしゃれにできる。日本人が個性を持てるようにする。

## 1. 中心構造 — 「世界観」という1本の軸

```
              ┌─────────────────┐
              │   世 界 観        │  ← すべての中心
              │ (その人だけの    │
              │  言語化された    │
              │  美意識・価値観) │
              └────────┬────────┘
                       │
   ┌──────────┬────────┼────────┬──────────┐
   ↓          ↓        ↓        ↓          ↓
 ① 知る     ② 変換    ③ 繋がる  ④ 発見     ⑤ 育てる
 診断       服/コーデ  投稿/     他者の     Knowledge
 世界観     /商品/    コミュニ  世界観/    OS が質を
 言語化     香水/音楽  ティ/     インスピ   上げ続ける
            /映画/    マッチング レーション (裏側エンジン)
            /アート
```

重要原則(オーナーの言葉):
> ただの寄せ集めではなく、すべてを「世界観」でつなげたい。

①〜⑤は独立機能ではない。全部「世界観」に刺さった団子。
どれか1つを主役にする設計は誤り。軸(世界観)が主役。

## 2. 中心体験の流れ(オーナー定義の理想フロー)

```
1. 診断する
2. 自分の世界観が言語化される
   →「自分ってこういう感覚を持っていたんだ」
   →「新しい自分を見つけた感じがする」
3. 世界観に合う 服・小物・香水・音楽・映画・アート が提案される
4. コーデとして「何をどう着ればいいか」分かる
5. 商品候補が表示される
6. 気に入ったものを購入できる(不利な状況・安い商品でも世界観を作れる)
7. 自分の世界観・コーデを投稿できる
8. 近い世界観の人・コミュニティと繋がれる
9. 他人の世界観からインスピレーションを得られる
```

この9ステップが「世界観」で1本に繋がっているのが理想。
現状(後述)は 1〜6 が部分的にでき、7〜9 が未着手。

## 3. 診断の本質(オーナーが繰り返し強調)

診断 =「似合う服を出す」ではなく「**新しい自分を知れる**」もの。

言語化したいもの:
- 好きな雰囲気 / 無意識に避けている印象 / 憧れている人物像 /
  本当はなりたい自分 / 服に求めている役割 / 価値観 / 世界観 /
  他人からどう見られたいか / どんな文化・空気感に惹かれているか

診断結果の構成(13項目・analyze-v2 で実装済み):
1. 世界観名 / 2. 無意識の傾向 / 3. 本当はなりたい自分 /
4. 避けている印象 / 5. 惹かれている文化・空気感 / 6. 合う色 /
7. 合う素材 / 8. 合うシルエット / 9. 合う小物 / 10. 合うブランド /
11. 合う音楽・映画・香水・アート / 12. まず試すべき服 /
13. 近い世界観の人・投稿・コミュニティ

→ 13番が「繋がる(コミュニティ)」への入口。診断とコミュニティは
  もともと地続きの設計だった。

## 5. 参考アプリの良い所を「世界観」でどう統合するか

| アプリ | 良い所 | STYLE-SELF での「世界観」化 |
|--------|--------|---------------------------|
| Pinterest | インスピ収集・ビジュアル発想 | ムードボード:世界観を視覚化。診断結果を画像で固める |
| Instagram | 自分を投稿・他人を見る・フォロー | 世界観の投稿。フォローは「数」でなく「世界観の近さ」 |
| TikTok | 発見性・興味でおすすめ | 世界観ベースのレコメンド。近い世界観の投稿が流れてくる |
| ZOZO等 | 服を探す・買う・比較 | 世界観に合う商品。安い物でも世界観を作れる提案 |

寄せ集めではなく、4つ全部を「世界観」という軸が貫く。
- Pinterest の収集 → 「自分の世界観のムードボード」
- Instagram の投稿 → 「自分の世界観の表現」
- TikTok の発見 → 「近い世界観との出会い」
- ZOZO の購買 → 「世界観の実体化」

## 6. タブ構成の提案

オーナー候補:
- 案1: ホーム / 探す / 投稿 / コーデ / 自分
- 案2: ホーム / 発見 / 買う / コーデ / 自分

評価(世界観軸で):
```
案1(探す/投稿) を推奨ベースに。理由:
- 「投稿」タブがあることで ③繋がる が構造に組み込まれる
- ビジョンの中心が「世界観で人と繋がる」を含むため、
  投稿への動線は一等地(タブ)にあるべき
- 「買う」をタブから外すのは正しい。商品は主役でなく
  各所(コーデ・世界観プロフィール)に内包される方が
  「世界観で繋がる」思想に合う

推奨タブ構成(調整版):
  1. ホーム    … 自分の世界観・今日の提案・近い世界観の新着
  2. 発見      … 他者の世界観/投稿/インスピ(Pinterest+TikTok的)
  3. 投稿      … 自分の世界観・コーデを投稿(Instagram的)
  4. コーデ    … 何をどう着るか(世界観×体型×季節×シーン)
  5. 自分      … 世界観プロフィール・診断・保存・購入履歴

「買う」は独立タブにせず、コーデ・世界観プロフィール・
発見の各所に「この世界観に合う商品」として埋め込む。
= 商品は世界観体験の一部であって、目的地ではない。
```

これは提案。次セッションでオーナーと最終決定する。

## 8. MVP と将来 — 何をどの順で作るか

### 大原則(オーナーの戦略・確定)
> 一旦全部の骨格を作る → そこから本/知識を入れて AI に育てさせる
> → 小さく当ててデカくする(収益も段階的)

骨格を「世界観で1本に繋がった状態」まで持っていくのが MVP の定義。

### MVP(骨格を繋ぐ・最小で一周させる)
```
M1. 診断を本番化(フェーズB Step5)= ①知る の完成
    ※レイテンシ Step6 はここに含めるか別途判断
M2. 世界観プロフィールを「人に見せられる」形に
    (現 DiagnosisDisplay を公開プロフィール化)
M3. 投稿機能の最小版(自分の世界観/コーデを投稿)= ③の基盤
M4. 世界観マッチングの最小版
    (近い世界観の人/投稿が見られる = 診断13番の実体化)
M5. 商品 worldview_tags 自動付与(②変換の空回り解消)
    ※商品ソース戦略の結論を待つ部分は最小限に
→ ここまでで「診断→世界観→提案→投稿→繋がる」が一周する
```

### 将来(骨格が一周した後・知識で育てる)
```
F1. ムードボード(世界観の視覚化)
F2. Knowledge OS 画像ナレッジ化(センス画像→判断ルール→コーデ精度)
F3. ビジュアルコンセプト評価軸を診断/コーデ/商品の頭脳に組み込む
F4. 商品ソース拡大(戦略決定後・質と量)
F5. レコメンド高度化(TikTok的発見性)
F6. コミュニティの深化(世界観グループ・イベント)
F7. 収益化(アフィリエイト→ブランド提携→サブスク、段階的)
F8. 「日本を変える」スケール(個性を持てる文化の醸成)
```

## 9. 商品ソース問題の正しい位置づけ(前回の混乱の整理)

```
前回「楽天もZOZOもいらない」で議論が紛糾した。
このマップで位置づけ直すと:

- 商品(②変換)は世界観体験の一部であって主役ではない
- だから商品ソース戦略は「今すぐ完璧に決める」必要はない
- MVP では worldview_tags 自動付与(空回り解消)だけやれば
  「世界観に合う商品が出る」状態にはなる(楽天の範囲でも)
- 「服好きが楽天で買わない」問題は F4(将来・戦略決定後)
- ZOZOアフィリエイトは2012年終了で存在しない(確定事実)
- 収益は F7。アフィリエイトは手段の1つにすぎず、
  最終的にはブランド提携・サブスクへ(オーナー戦略)

結論: 商品ソースは MVP のブロッカーにしない。
      worldview_tags 自動付与(M5)だけ MVP に入れ、
      ソース拡大・収益は骨格が一周してから(F4/F7)。
```
