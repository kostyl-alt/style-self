## 1. ★ オーナービジョン verbatim 記録(★ 改変禁止)

### 1.1 目的

```
ユーザーがムードボード画像やテキストを入力すると、
AI がそのムードボードを世界観要素に分解し、
ユーザーの診断結果・保存行動・クリック行動と組み合わせて、
世界観に合うコーデ・商品・投稿・コミュニティを提案する仕組み。
```

### 1.2 考え方(★ 三位一体)

```
- 診断結果       = ユーザーの基本世界観
- ムードボード   = 今回作りたい空気感・一時的な方向性
- 保存/クリック/いいね/購入/投稿 = 実際の好みの証拠
```

### 1.3 実装したい流れ(★ 9 ステップ)

```
1. MB 画像複数枚アップロード
2. 自由記入(どこが好き/どう見られたい/避けたい印象/着るシーン/季節/予算)
3. AI が MB 分析
4. 分析項目(14):中心ムード/色/素材/シルエット/比率/重心/ライン/構造/調和/機能/テーマ/
                文化参照/避けるべき要素/コーデに変換できる要素
5. 診断結果の世界観プロフィールと MB 分析を照合
6. コーデ提案生成
7. 商品提案生成
8. なぜこのコーデが MB と本人世界観に合うか説明
9. 保存/クリック/再提案/購入行動を記録して次回推薦に反映
```

### 1.4 重要(★ 設計思想の核心)

```
- MB 画像の表面だけを真似しない
- 黒い画像だから黒い服、花だから花柄、の浅い変換は禁止
- 画像の奥のムード/距離感/緊張感/素材感/光/構図/文化参照を抽出
- 診断結果と矛盾する場合は完全に寄せず「本人に似合う範囲で日常化」
- コーデ説明は 11 項目(比率/素材/色/カット/シルエット/ライン/重量/構造/調和/機能/テーマ)
```

### 1.5 出力モード(★ 3 モード)

```
- 忠実モード:   MB に強く寄せる
- 日常化モード: 普段着に落とす
- 挑戦モード:   本人の世界観を広げる
```

### 1.6 DB 設計案(★ 7 テーブル)

```
moodboards / moodboard_images / moodboard_analysis /
user_worldview_profiles / user_actions / outfit_recommendations / recommendation_feedback
```

### 1.7 UI 表示

```
MB 分析結果 / 今回のコーデ方針 / 提案コーデ / 商品候補 / なぜ合うか /
どの要素が MB 由来 / どの要素が診断結果由来 /
保存・好き・違う・もっと寄せる・日常化する フィードバックボタン
```

## 5. ★ 「表面を真似しない」中核思想記録(★ STYLE-SELF を唯一無二にする原則)

### 5.1 ★ 原則の明文化

```
★ MB 画像の表面だけを真似しない:
- 黒い画像 → 黒い服 ❌(浅い変換)
- 花の画像 → 花柄 ❌(浅い変換)

★ 画像の奥を抽出する:
- ムード(感情・空気感)
- 距離感(被写体との関係)
- 緊張感(構図の張り)
- 素材感(光の反射 / 表面の質)
- 光(自然光 / 人工光 / 時間帯 / 強度)
- 構図(三分割 / 対称 / 黄金比)
- 文化参照(美術運動 / 映画 / 音楽 / 時代)
```

### 5.2 既存 buildMoodboardPrompt(`2e10587`)との関係

現状の `buildMoodboardPrompt` 出力:
```
コンセプト: 25 歳富裕旅行者 / 海岸 / 夕方
[必須要素 8] + 参考画像メモ
【応答形式の希望】11 項目 + 7 アイテム
```

→ ★ **「表面を真似しない」原則は ★ 既に部分的に体現**(11 項目 + 「不明な要素は世界観から推定」)。

### 5.3 完全化(★ Sprint E-1 領域)

★ Sprint E-1 で 14 項目構造化時に強化:
```
[MB 分析(★ 表面ではなく奥を抽出)]
1. 中心ムード(例: 静謐 / 緊張感 / 孤独 / 多幸感)
2. 色(★ 印象としての色・特定色を強制しない)
3. 素材(質感・光沢段階)
4. シルエット
5. 比率
6. 重心
7. ライン
8. 構造(完成 / 解体 / 過剰)
9. 調和(集約 / 拡散)
10. 機能(動き / 静止 / 装飾 / 実用)
11. テーマ(時代 / 場所 / 概念)
12. 文化参照(美術運動・映画・音楽)
13. 避けるべき要素(★ 画像から読み取れる「これは違う」)
14. コーデに変換できる要素(★ Why & How 橋渡し)
```

### 5.4 「本人に似合う範囲で日常化」原則

★ 診断結果との矛盾時:
- 完全に寄せない(MB の世界観 100% 再現ではない)
- 本人の体型 / 好み / 避けたい印象を尊重
- 3 モードで対応:
  - 忠実モード: MB に寄せる(矛盾は本人尊重で部分調整)
  - 日常化モード: 普段着に落とす(MB はインスピレーション)
  - 挑戦モード: 本人世界観を広げる(MB が手引き)

→ ★ STYLE-SELF を「Pinterest クローン」ではなく「**自分専用 AI スタイリスト**」にする原則。
