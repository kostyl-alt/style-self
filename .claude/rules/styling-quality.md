# スタイリング品質ルール（AI出力の質を構造で担保する）

このプロジェクトの AI 出力（コーデ・分析・vision）は、以下を**構造で**満たす。
プロンプトで効かないと分かったら構造（コード側の決定的処理）に倒す。

## 1. 事実ベースで語る
次の観点を事実として扱い、根拠なく飾らない:
色 / 素材 / 丈 / 比率（上下バランス）/ 重心 / 小物 / 季節性 / 身体相性。

## 2. 写っていないことを断定しない
画像・データから確実でない値は `basis: inferred` として推測と明示する（確実なものは `observed`）。
素材・ヘアメイク・体型適性などは多くが inferred 前提。

## 3. 構造を再現する
コーデは「なんとなく似た雰囲気」でなく、色・シルエット・比率・重心の**構造**を再現する。

## 4. ロングコートは既定で出さない
ユーザー方針として、コーデ提案でロングコートを既定で出さない。
例外が必要な場合はユーザーが明示した時だけ扱う。

## 5. 身体相性は断定でなく傾向で言う
- ❌「骨格ストレート向き」
- ✅「骨格ストレート傾向なら成立しやすい」
身体相性は個人差があるため、断定でなく「傾向なら成立しやすい」という弱い言い方にする
（ルール本文「断定しない」と例文の一貫性を保つ）。

## 6. 「買える条件」まで落とす
世界観で止めず、買い物に使える条件まで具体化する:
ジャンル / 年代 / カルチャー / 素材 / 色 / シルエット。

## 7. 固有名・国籍を捏造しない
ブランド名や国籍を LLM に作らせない。
ブランドカード本体は `lib/knowledge/brand-match.ts`（`matchBrands`）がコードで決定的に組み、
LLM は前置き/締めの自然会話だけを担当する。タグは `lib/style-taxonomy.ts` の `STYLE_AXES` 実在語に正規化する。
